# 구현 정리

이 문서는 20x20 미로에서 기존 A*와 압축 그래프 기반 A*를 비교하는 프로그램을 어떤 방식으로 만들었는지 정리한 기록이다.

## 1. 미로 생성

미로는 `maze.py`에서 생성한다. 크기는 기본값으로 20x20을 사용한다.

미로 문자는 다음과 같이 통일했다.

- `0`: 지나갈 수 있는 공간
- `1`: 벽
- `2`: 시작점
- `3`: 끝점

미로 생성은 DFS 방식으로 진행했다. 처음에는 모든 칸을 벽으로 채운 뒤, `(1, 1)`에서 시작해 두 칸 단위로 이동하며 중간 벽을 뚫는다. 이렇게 하면 연결된 미로가 만들어지고, 시작점에서 끝점까지 항상 경로가 존재한다.

생성된 미로는 `maze/` 폴더에 저장된다. 파일명은 소문자 알파벳과 숫자로 이루어진 16자 랜덤 문자열이며, 같은 이름이 이미 있으면 다시 생성한다.

## 2. 기존 A* 구현

기존 A*는 `astar.py`에 구현했다.

이 방식은 20x20 미로의 모든 이동 가능한 칸을 노드로 본다. 각 칸은 상하좌우 네 방향으로 이동할 수 있고, 벽인 칸은 이동할 수 없다.

A*의 우선순위는 다음 식으로 계산한다.

```text
f(n) = g(n) + h(n)
```

- `g(n)`: 시작점에서 현재 칸까지 실제 이동한 거리
- `h(n)`: 현재 칸에서 도착점까지의 예상 거리
- `f(n)`: 탐색 우선순위

휴리스틱 함수는 4방향 격자 미로에 맞게 맨해튼 거리를 사용했다.

```text
h(n) = |row - goal_row| + |col - goal_col|
```

## 3. 압축 그래프 생성

압축 그래프는 `compressed_graph.py`에서 만든다.

기본 아이디어는 미로의 모든 칸을 탐색 대상으로 삼지 않고, 경로 선택에 영향을 주는 중요한 지점만 노드로 남기는 것이다. 두 중요한 노드 사이의 단순한 통로는 하나의 가중치 간선으로 압축한다.

처음 PDF 설계에 맞춘 `design` 모드에서는 다음 칸들을 노드로 사용한다.

- 시작점
- 끝점
- 갈림길
- 꺾이는 지점
- 막다른 길

이후 성능을 더 개선하기 위해 `optimized` 모드를 추가했다. 이 모드에서는 꺾이는 지점이라도 이동 가능한 방향이 정확히 2개인 단순 통로라면 노드로 남기지 않고 간선 안에 포함한다. 이렇게 해도 실제 이동 거리는 간선 가중치로 보존되므로 최단 경로 길이는 변하지 않는다.

## 4. 압축 그래프 기반 A*

압축 그래프 기반 A*는 `cga_star.py`에 구현했다.

기존 A*와 같은 방식으로 `f(n) = g(n) + h(n)`를 사용하지만, 탐색 대상이 격자의 모든 칸이 아니라 압축 그래프의 노드이다.

압축 그래프의 간선은 실제 복도 길이를 가중치로 가진다. 예를 들어 두 노드 사이에 이동해야 하는 칸 수가 8이면 간선 비용도 8이다.

탐색이 끝나면 압축 그래프의 노드 경로를 다시 실제 미로 칸 경로로 펼친다. 그래서 결과 경로는 기존 A*와 같은 형식으로 비교할 수 있다.

## 5. 비교 실험

비교 실험은 `compare.py`에서 실행한다.

각 실험은 같은 미로에 대해 다음 두 알고리즘을 모두 실행한다.

1. 기존 격자 기반 A*
2. 압축 그래프 기반 A*

비교 지표는 다음과 같다.

- 탐색한 노드 수
- 최단 경로 길이
- 실행 시간
- 탐색 노드 감소율
- 실행 시간 감소율

압축 그래프 방식이 올바른지 확인하기 위해, 두 알고리즘의 최단 경로 길이가 다르면 오류가 발생하도록 했다. 즉, 탐색 효율은 좋아져도 경로 품질이 나빠지는 경우는 실패로 처리한다.

## 6. 시각화

시각화는 `visualize.py`에서 SVG 파일로 만든다. 별도 라이브러리 없이 브라우저와 GitHub에서 바로 볼 수 있게 하기 위해 SVG를 선택했다.

생성되는 주요 시각화 파일은 다음과 같다.

- `results/latest_maze.svg`: 미로, 최단 경로, 압축 그래프 노드 표시
- `results/latest_comparison.svg`: 기존 A*와 압축 그래프 A*의 평균 탐색 노드 수 및 실행 시간 비교

미로 시각화의 색상 의미는 다음과 같다.

- 검정: 벽
- 흰색: 이동 가능 공간
- 파랑 선: 최단 경로
- 초록 `S`: 시작점
- 빨강 `E`: 끝점
- 주황 점: 압축 그래프의 탐색 노드

## 7. 실행 방법

미로 하나를 생성하려면 다음 명령을 사용한다.

```powershell
python .\main.py generate
```

30개의 미로에 대해 최적화된 압축 그래프 방식으로 비교하려면 다음 명령을 사용한다.

```powershell
python .\main.py compare --runs 30 --seed 1578 --mode optimized
```

PDF 설계와 같은 기준으로 꺾이는 지점까지 노드로 남기려면 다음 명령을 사용한다.

```powershell
python .\main.py compare --runs 30 --seed 1578 --mode design
```

## 8. 최신 결과

최신 `optimized` 모드 30회 실험에서는 모든 실험에서 기존 A*와 압축 그래프 A*의 최단 경로 길이가 같았다.

요약 결과는 다음과 같다.

- 평균 기존 A* 탐색 노드 수: `77.27`
- 평균 압축 그래프 A* 탐색 노드 수: `6.97`
- 평균 탐색 노드 감소율: `91.31%`
- 평균 기존 A* 실행 시간: 약 `0.15 ms`
- 평균 압축 그래프 A* 실행 시간: 약 `0.01 ms`

따라서 이 구현에서는 최단 경로 길이를 유지하면서 탐색해야 하는 노드 수를 크게 줄일 수 있었다.
